# CLAUDE.md

Project guidance for Claude Code working in this repo.

## Communication style
When summarizing progress or explaining what was done, default to plain, non-technical
language — pitch it like a short **elevator pitch to a non-technical VC**: what it does, why
it matters, and what's next. Lead with the outcome and the benefit, not the implementation;
keep jargon out unless detail is requested. The user will explicitly ask for technical depth
(diffs, file paths, commands) when they want it — until then, keep it simple and scannable.
